# 画面遷移・メニュー設計

Kudamono Battle の現時点の画面構成と、今後キャラクターやステージを増やすための設計方針です。

## 画面フロー

基本の遷移は次の通りです。

```text
Title -> Character Select -> Stage Select -> Game -> Result
```

結果画面からは、次の3つの遷移を用意しています。

| 操作 | 遷移先 | 用途 |
| --- | --- | --- |
| `R` / REMATCH | Game | 同じキャラクターとステージでもう一度遊ぶ。 |
| `C` / CHARACTER SELECT | Character Select | 現在の選択状態を引き継いだままキャラクター選択からやり直す。 |
| `T` / TITLE | Title | タイトル画面へ戻る。 |

## タイトル画面

タイトル画面は、生成画像のキービジュアル `public/assets/ui/title-key-art.webp` を背景に使います。ロゴ文字は画像に焼き込まず、Phaser のテキストで重ねています。

この方針にしている理由:

- タイトル文字を後から修正しやすい。
- 多言語化やロゴ差し替えに対応しやすい。
- 画像生成時の文字崩れを避けられる。

タイトル画面の主な役割:

- ゲームの雰囲気を伝える。
- `START` でキャラクター選択へ進める。
- キーボードでは `Enter` でも進める。

## キャラクター選択画面

キャラクター選択はカード型 UI です。現在は2体だけですが、後からキャラクターを追加しやすいように、データ配列からカードを生成します。

キャラクターカードに表示する情報:

- キャラクター選択用ポートレート。
- キャラクター名。
- 役割。
- 簡単な特性説明。
- 選択中マーカー。

現在のキャラクター:

- Strawberry Samurai
- Banana Brawler

現在は Player 1 がキャラクターを選び、CPU はライバル側の果物を自動選択します。将来的に CPU 側や2P側の選択を追加する場合も、同じカード UI を流用できます。

## ステージ選択画面

ステージ選択もカード型 UI です。現在は `Kitchen Counter` の1ステージだけですが、ステージデータを追加すればカードを増やせる構成にしています。

ステージカードに表示する情報:

- ステージプレビュー画像。
- ステージ名。
- サブタイトル。
- 簡単な説明。
- 選択中マーカー。

現在のステージ:

- Kitchen Counter / Cutting Board Clash

## MatchConfig

ゲーム開始時には、選択結果を `MatchConfig` として `GameScene` に渡します。

現在の構成:

```ts
{
  playerCharacter: 'strawberry' | 'banana',
  cpuCharacter: 'strawberry' | 'banana',
  stage: 'kitchen'
}
```

`ResultScene` にも同じ `MatchConfig` を渡すことで、リマッチ時に同じ条件で再開できます。

## 今後の拡張方針

- キャラクター追加時は、キャラクター定義、ポートレート、モーション画像、技データを追加する。
- ステージ追加時は、ステージ背景、プレビュー、足場判定、スポーン位置を追加する。
- キャラクター選択画面は、カード数が増えたら横スクロールまたはページ切り替えにする。
- ステージ選択画面は、ステージ数が増えたらグリッド化する。
